Memory Systems

The architectural component responsible for persistence, retrieval, and contextual continuity across interaction sessions.

psychology

Mental Model

Think of the Memory System as a librarian with a short-term notepad. It maintains a vast, indexed archive (Vector DB) while keeping immediate sticky notes (Context Window) for the conversation at hand.

What this is

A composite system integrating vector-based semantic search with a sliding-window buffer. It decouples information storage from model weights, allowing the agent to "remember" user preferences and factual constraints dynamically without retraining.

Why it matters

It prevents catastrophic forgetting in long-running tasks. Without this layer, the agent operates in a perpetually amnesic state, unable to reference decisions made even a few minutes prior once they slide out of the immediate token window.

Design Tradeoffs

  • Latency vs. Recall Full semantic search adds ~200ms overhead per turn, creating a noticeable pause in real-time interactions compared to pure context-window lookup.
  • Precision vs. Fuzzy Matching Vector embeddings capture meaning but struggle with exact keyword retrieval (e.g., specific IDs), necessitating hybrid search strategies.
  • Cost vs. Context Depth Storing infinite history scales storage costs linearly, forcing the implementation of aggressive summarization or decay algorithms that may lose nuance.
